Doc lens is for conversing with your PDF documents. The documents can be text pdfs images, the document can even have charts, it will recognize them and make use of that information.
When a user uploads the document, first it is converted to chunks which are page bound,i dont remember more than this. for this we have used docling as it can handle ocr as well. for charts and images etc we use a second modal, the main concept multimodal, to work with the images and tables charts etc to make use of those details instead of simply extracting text details, when docling returns 0 text ectracted then the ocr or the second modal runs.  After that these chunks are converted to embeddings by using bge small model as it is free and decent. then these embeddings are stored in the vector database Qdrant. then the user can ask questions related to the document. The document is processed parallel using celery and redis celery distributes the task and redis manages i think. this is done so that the usewr doesnt have to wait and can ask questions. the query also is embedded and then we look it up for similarity i think we use cosine similarity search as wmbeddings are vectors and we select on similar planes etc.similar chunks are retrieved by the llm modal api and answer is generated.

we use fastapi to request from frontend to backend models and get responses on different routes. the Frontend UI is made using react i think or simple HTML CSS JS
